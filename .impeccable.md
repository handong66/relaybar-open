## Design Context

### Users
Developers and power users who keep multiple Codex accounts on one Mac. They open the menu bar panel for a quick quota check, account switch, and recent usage glance without leaving their current workflow.

### Brand Personality
Compact, calm, exact. The interface should feel like a polished macOS utility: confident, low-noise, and fast to scan.

### Aesthetic Direction
Light mode first. Native macOS panel proportions, restrained tinting, minimal chrome, and tight information density. Use OpenUsage as a reference for compact section rhythm and small-but-readable typography. Avoid bulky cards, oversized spacing, or dashboard-like visual weight.

### Design Principles
1. Keep the panel small enough to feel like a menu bar utility, not a standalone dashboard.
2. Prioritize glanceability: labels should be short, progress bars compact, and reset info immediately adjacent.
3. Use emphasis sparingly. Active state should be clear but not flood the panel with color.
4. Secondary metrics such as credits and local usage should read as lightweight summaries, not dominate the card.
5. Preserve macOS utility polish: subtle borders, rounded geometry, soft shadows, and consistent control sizing.
